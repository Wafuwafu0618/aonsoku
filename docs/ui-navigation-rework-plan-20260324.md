# UI Navigation Rework Plan (2026-03-24 JST)

## 背景
現行UIは機能は揃っているが、左ナビゲーションの区分と利用導線が弱く、商業アプリとしての使い心地に不足がある。  
特に「使われないタブ」と「ホームの情報不足」が体験のボトルネック。

## ユーザー観点の不満（整理）
- 左タブの区分が実利用に対して過多/不一致。
- 「曲」タブが全曲1列表示中心で、利用価値が低い。
- ホームのセクション（最近再生/よく再生/最近追加/探索）が物足りない。
- プレイリスト構造は現状維持で問題なし。
- Apple Music との連動は強化したいが、UI上は自然に統合したい。

## 方針（確定）

### 1) 左タブ再編
メインナビを以下に固定:
1. 検索
2. ホーム
3. ラジオ
4. ライブラリ
5. プレイリスト

補足:
- `進捗` タブは作らない（再起動復元や下部プレイヤー表示で代替できるため）。
- 「曲」をメインタブから外し、ライブラリ配下の1カテゴリとして扱う。

### 2) ライブラリの定義を明確化
`ライブラリ` は「お気に入り追加済みの項目のみ」を対象にする。  
配下カテゴリ:
1. 最近追加（お気に入り追加順）
2. アーティスト
3. アルバム
4. 曲
5. おすすめ

### 3) ホーム強化
ホームは「再生履歴の羅列」ではなく、発見性を持たせる。  
最低限の拡張:
- ジャンル別おすすめセクション
- 利用履歴に基づく再提案（最近再生/よく再生の改善版）

## Apple Music 連動方針（UI/UX）

### 原則
- UIは統合、データソースは分離管理。
- Apple Music を特別扱いしすぎず、同一体験内に自然統合する。

### 実装ポリシー
1. 検索は横断検索を基本（必要時にソースフィルタ）
2. ライブラリは「お気に入り追加済み」のみ取り込む
3. プレイリストは現状構造を維持
4. おすすめは `履歴 + お気に入り + ジャンル + ソース` で混成

## 実装優先順（フロントエンドフェーズ）
1. Sidebar IA再編（文言/順序/グルーピング/route）
2. ライブラリ画面を「お気に入り限定モデル」に統一
3. ホームへジャンル別おすすめブロック追加
4. Apple Music をおすすめ生成と検索結果へ統合
5. 表示密度・余白・階層コントラストの調整（商業品質チューニング）

## 受け入れ基準（暫定）
- 左ナビで実利用導線が `検索 / ホーム / ラジオ / ライブラリ / プレイリスト` に整理される。
- 「曲」単独タブがなくても、ライブラリ内導線で同等以上に到達しやすい。
- ホームに「発見系」の価値（ジャンルおすすめ）が追加される。
- Apple Music を含む横断体験でも、操作負荷が増えない。
